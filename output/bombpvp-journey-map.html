How to use Journey Maps

What this is

An NN/g customer journey map and service blueprint combined into a single grid — one per user journey. The top half is the customer view; the bottom half is the service layer. Both share the same columns (stages and moments).

Navigating journeys

Select any journey from the left rail. Journeys are grouped by category.

Reading the grid

Columns are stages, each containing one or more moments. Within each moment column, rows are stacked top-to-bottom:

Doing Thinking Feeling Pain points Opportunities Evidence

A line of interaction separates the customer journey from the service blueprint. Below it:

Frontstage Backstage Support processes

What you can edit

  • Scenario & User goal — editable in the journey header. Your changes shadow the built-in defaults.
  • Thinking — free-text per moment. What is the user asking themselves at this point?
  • Feeling — pick one emoji per moment. Drives the emotion curve below the grid.
  • Pain points & Opportunities — your research findings. One item per line works well.
  • Evidence — link to a source: an insight, a session recording, a ticket ID.
  • Notes — a full-width freeform textarea below the grid for anything that doesn't fit a cell.

The Doing, Frontstage, Backstage, and Support rows are pre-filled and read-only in the UI. Their content lives in the JSON block at the bottom of the file; edit the JSON directly to change them.

Emotion curve

As soon as two or more moments in a journey have a feeling set, a curve renders automatically below the grid. It traces sentiment across stages so you can spot where the journey dips.

Saving & sharing

  • Autosave — every edit saves to this browser's localStorage after a 350 ms pause. The rail shows "Saved" when it commits.
  • Export — downloads a timestamped JSON of all annotations. Back it up or send it to a teammate.
  • Import — loads a JSON file. You'll be asked whether to replace your local state or merge a colleague's annotations into yours (safe for collaboration).
